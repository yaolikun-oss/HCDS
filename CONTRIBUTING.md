# Contributing to HCDS

Thank you for your interest in contributing to HCDS.

HCDS is maintained as a software standard. Contributions should preserve the frozen architecture and avoid changing specifications unless a specification proposal has been accepted.

## Contribution Types

- Documentation fixes
- Repository infrastructure improvements
- Tooling, validators, and generators
- Test coverage
- Specification proposals through RFCs or issue templates

## Before Opening a Pull Request

- Keep changes focused and small.
- Do not rename frozen documents.
- Do not change architecture decisions without an accepted proposal.
- Add or update tests when changing tools, generators, validators, or CI.
- Update documentation indexes when adding documentation.

## Specification Changes

Specification changes must begin as a Specification Proposal issue or RFC. Do not submit direct specification edits without prior discussion.
